Thanks for your interest in contributing!

Guidelines:
- File issues for bugs or feature requests.
- Fork the repo and open a pull request for changes.
- Keep PRs small and focused.
- Add tests for new behavior where applicable.

Code style:
- Kotlin with standard formatting. Run ktlint or the project's code style checks before opening PRs.

Security:
- Do not include secrets or credentials in PRs or issues. If you must disclose a vulnerability, use the process in SECURITY.md.
